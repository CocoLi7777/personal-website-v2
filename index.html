COVID-19 status data visualization in Australia

This is a side project to help people visualize the current COVID-19 trending in Australia. This web application consumes an API from the community and did some transformation to display the overall trending as well as the trending in each state.

Live demo: https://c19-trending.herokuapp.com/

Staging environment: https://c19-trending-stg.herokuapp.com/

Source code: https://github.com/CocoLi7777/COVID-19-trending

Highlights and technology:

Front end written in vanilla JavaScript to reduce the script size and for better performance. Use node express to build a proxy for the front-end and serve the static files. Responsive UI using Bulma CSS framework, mobile friendly. Using Chart.js to visualize the data in different formats, like pie chart and bar chart. Setup Continuous Deployment using Github and Heroku, seamless development experience from source code to production. Use Git branch development for the workflow, separate the dev and release branch and deployment pipeline for staging and production environment. Use Google Analytics to track the website usage Integrate with Slack webhook to notify any production error logs in order to resolve the issue in a timely manner. Build local file system cache for the data daily for the backup plan for the third party API failure.

Supermarket stock availability automation

The idea of this project comes in the current situation to fight COVID-19 pandemic. As the panic buying disrupts supermarket stock and distribution, I would like to create a tool that can help people to receive the availability notification easier and avoid unnecessary movement. This project is built with Cypress to run an UI automation to check the supermarket product availability. With the help of the CircleCI scheduled workflow, it will be running periodically to help people check the stock availability for a particular product and receive Slack notification. This is a useful tool that can help people use it on a daily basis, also it is a good practice to try out the end to end testing framework and continuous integration tools. Most importantly, this project is a simple, but not easy, solution to solve real life problems in a cost-effective(no cost) manner.

Source Code: https://github.com/CocoLi7777/supermarket_auto

Highlights and technology:

UI automation with Cypress Use CircleCI to run the job to check the availability Setup scheduled cron job on for the CI workflow Integrate Slack for the notification of the availability result
